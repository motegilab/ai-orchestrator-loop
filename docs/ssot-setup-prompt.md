## あなたがやること（順番通りに）

### Step 1 — プロジェクトの情報をヒアリングする

次の質問を一度にまとめて聞いてください（一つずつでなくまとめて）:

1. このプロジェクトは何を作るもの？（一言で）
2. 使う技術スタックは？（例: Python + FastAPI + SQLite）
3. 最終的に「これができたら完成」という状態は？
4. まず最初に作りたい機能・モジュールは何？

---

### Step 2 — SSOT.md を生成する

ヒアリング結果をもとに、以下の構造で `SSOT.md` の内容を生成してください。
生成したら「これをコピーして SSOT.md に貼り付けてください」と渡す。

```markdown
# SSOT.md — [プロジェクト名]

## §0 設計原則
（プロジェクトの方針・大切にすること 3〜5 箇条）

## §1 絶対ルール
（ループ中に AI が必ず守るルール。例: テスト未 PASS の状態でコミットしない）

## §2 環境スタック
（OS / 言語バージョン / フレームワーク / DB / ツール一覧）

## アーキテクチャ
（コンポーネント構成と依存関係の図または箇条書き）
```

SSOT.md を保存したら、必ずこのコマンドを実行するよう伝える:
```bash
make setup
```

---

### Step 3 — tasks/milestones.json を生成する

プロジェクト内容をもとに、最初の 1〜2 マイルストーン分のタスクを生成してください。

**タスク設計のルール（守らないとループがブロックされる）:**
- タイトルは **20 文字以上** で「何をする・何を確認する」が明確なこと
- `TBD` / `未定` / `TODO` は ERROR になるので絶対に使わない
- 各マイルストーンに `verify` か `pytest` か `確認` を含むタスクを最低 1 つ入れる
- 最初の人間確認ポイントには `"checkpoint": true` を付ける

**JSON 構造:**
```json
{
  "project": "プロジェクト名",
  "milestones": [
    {
      "id": "M1",
      "title": "Phase 1: フェーズ名（具体的に）",
      "status": "pending",
      "waves": [
        {
          "id": "W1-1",
          "title": "ウェーブ名（実装するコンポーネント名など）",
          "tasks": [
            {
              "id": "T1",
              "title": "20文字以上の具体的なタスクタイトル",
              "status": "pending"
            },
            {
              "id": "T2",
              "title": "verify: pytest で全テスト PASS を確認する",
              "status": "pending",
              "checkpoint": true
            }
          ]
        }
      ]
    }
  ]
}
```

生成したら「これをコピーして tasks/milestones.json に貼り付けてください」と渡す。

---

### Step 4 — 最初のループを起動する

milestones.json の保存を確認したら、以下を案内する:

```bash
# ⚠️ VSCode の統合ターミナルではなく、外部の PowerShell か Git Bash から実行すること
make loop-run
```

起動後のプロンプトへの答え方を教える:
```
SSOT チェックをやりますか? [Y/n]: y    ← Enter か y を押す
ループ回数は最大何回にしますか?: 3     ← まず 3 回で様子を見る
```
