?このシステムとは

AI Orchestrator Loop は、Claude Code CLI をエンジンとした自律実行ループです。 タスク定義ファイル（tasks/milestones.json）に "何をすべきか" を書いておくだけで、 Claude が Observe → Patch → Verify → Report のサイクルを自動で繰り返し、タスクを消化していきます。

Observe
問題・タスクを調査。Evidence first で現状把握
Patch
最小差分で修正。1 原因 1 修正を厳守
Verify
exit code と stdout を確認。失敗は次ループへ
Report
結果を REPORT_LATEST.md に記録。Stop Hook が次のコンテキストを生成

3Discord 通知を設定する（任意）

ループの開始・完了を Discord に通知できます。通知不要なら Enter でスキップしてかまいません。設定したい場合は下の手順で Webhook URL を取得してください。

① Discord で Webhook URL を取得する

  1. Discord を開き、通知を受け取りたい サーバー を選ぶ（なければ新規作成）
  2. 通知を送りたい テキストチャンネル を右クリック → チャンネルの編集
  3. 左メニューの 連携サービス（Integrations）を開く
  4. ウェブフック（Webhooks）→ 新しいウェブフック をクリック
  5. 名前を好きに変えて（例: AI Loop）、ウェブフック URL をコピー を押す
  6. コピーした URL は https://discord.com/api/webhooks/1234.../xxxx... という形式
💡
チャンネルが見つからない場合：サーバー設定 → 連携サービス → ウェブフック からも作成できます。

② notifications.json に貼り付ける

auto_setup.py で聞かれた URL 入力欄に貼り付けるか、以下のように直接ファイルを編集します。

// notifications.json（プロジェクトルートにあります）
{
  "discord": {
    "enabled": true,
    "webhook_url": "https://discord.com/api/webhooks/YOUR_ID/YOUR_TOKEN",
    "notify_on": ["stop"],
    "mention_role_id": ""
  }
}

YOUR_ID/YOUR_TOKEN の部分をコピーした URL 全体で丸ごと置き換えてください。

③ セットアップ後に設定し直す場合

auto_setup.py をスキップした後でも、notifications.json をテキストエディタで直接開いて URL を書き換えれば有効になります。

$ python tools/scripts/auto_setup.py もう一度実行すると再設定できます
✅
notifications.json は .gitignore で除外済みです。Webhook URL がリポジトリに公開される心配はありません。

5tasks/milestones.json を定義する

ループが消化するタスクを定義します。

{
  "project": "my-project",
  "milestones": [
    {
      "id": "M1",
      "title": "Phase 1: 初期実装",
      "status": "pending",
      "waves": [
        {
          "id": "W1-1",
          "title": "コア実装",
          "tasks": [
            {
              "id": "T1",
              "title": "20文字以上でタスクを具体的に書く",
              "status": "pending"
            },
            {
              "id": "T2",
              "title": "verify: pytest で全テスト PASS を確認する",
              "status": "pending",
              "checkpoint": true   // ← ここで loop-run が一時停止
            }
          ]
        }
      ]
    }
  ]
}

タスク設計のルール

  • ✓ タイトルは 20 文字以上 で具体的に（短すぎると ssot-check が警告）
  • ✓ 各マイルストーンに pytest / verify を含むタスク を最低 1 つ入れる
  • ✓ TBD / 未定 / TODO はエラーになるので使わない
  • ✓ コンポーネント単位でウェーブを分けると Claude の精度が上がる
  • ○ 中間確認が必要な箇所に "checkpoint": true を付ける（任意）

↻ループの内部動作

make loop-run
  ↓
ssot_check.py  ← SSOT に重大な問題があればブロック
  ↓
loop_start.py  ← Claude Code CLI を起動
  ↓
  【Claude セッション】
    SessionStart Hook  ← next_session.md を Claude に注入
    observe Skill      ← 問題・タスクを調査
    patch Skill        ← 最小差分で修正
    verify Skill       ← exit code と stdout を確認
    report Skill       ← REPORT_LATEST.md に記録
    Stop Hook          ← on_stop.py が次セッション情報を生成
  ↓
loop_run.py が次タスクを確認  ← pending があれば繰り返し

生成されるファイル

ファイル役割

runtime/logs/next_session.md次セッションへの引き継ぎ指示

✓SSOT チェックの仕組み

make loop-run の前に自動実行されます。make ssot-check で単体実行も可能。

判定条件loop-run への影響
🔴 ERROR§0/§1/§2 欠如、TBD/未定タスクあり、JSON 破損ブロック（--skip-check で強制実行可）
🟡 WARN統合タスクなし、タイトル短すぎ、依存ルール未定義続行確認 [y/N]
🟢 OK問題なし即ループ開始
=== SSOT Quality Check ===
    OK    §0 設計原則: 存在
    OK    §1 絶対ルール: 存在
    OK    §2 環境: 存在
   WARN   M2 に統合確認タスク（pytest/verify）がありません
  ERROR   T5 タイトルに未定義キーワード: 「TBD: あとで実装」

スコア: 45/100
判定: ERROR — SSOT に重大な問題があります

⏸チェックポイント機能

特定タスクの手前で loop-run を自動停止させる機能です。

{
  "id": "T5",
  "title": "統合テスト: 全コンポーネントが連携して動くことを確認する",
  "status": "pending",
  "checkpoint": true   // ← このタスクを実行する前に停止
}

停止時のメッセージ:

[loop-run] チェックポイント: T5 — 統合テスト: 全コンポーネントが連携して...
[loop-run] このタスクの前で停止します。確認後 make loop-run で再開してください。

確認後は make loop-run で再開するだけです。チェックポイントを通過したタスクは自動的に続行します。

!トラブル対処

Stop Hook が発火しない / runtime/runs/ にファイルが作られない

⚠️
VSCode 統合ターミナルから make loop-start を実行すると、Claude Code のサブプロセスとして起動され Stop Hook が正常に発火しません。外部の PowerShell か Git Bash から実行してください。

SSOT.md を編集しようとしてブロックされた

正常な動作です。SSOT.md はループ中の自動編集が禁止されています。手動で編集した後 make setup を実行してください。

make setup が失敗する

以下のコマンドで詳細なエラーメッセージを確認してください。

python tools/scripts/setup.py

赤いエラーメッセージが出たら、その文字を丸ごとコピーして Claude に「このエラーが出ました。原因と対処法を教えてください」と聞いてください。

claude: command not found

Claude Code がインストールされていません。以下の順番で実行してください。

# ① Claude Code をインストール
npm install -g @anthropic-ai/claude-code

# ② バージョン確認（v2.0 以上なら OK）
claude --version

# ③ もう一度セットアップを実行
make setup

連続 incomplete でループが止まった

Claude が report Skill を実行せずにセッションを終了しました。以下の手順で確認・再開してください。

# ① 前回の状態を確認
make loop-status

# ② レポートをテキストエディタで確認（VSCode の場合）
code runtime/reports/REPORT_LATEST.md

# ③ エラーログを確認（最後の数行を見る）
python -c "lines=open('runtime/artifacts/audit_log.jsonl').readlines(); [print(l) for l in lines[-5:]]"

# ④ 内容を確認したら再実行
make loop-run

エラーが読み取れない場合は runtime/artifacts/audit_log.jsonl をVSCodeで開いて最後の行を Claude に見せてください。

↑テンプレートの更新を自分のPJに反映する

テンプレートリポジトリ（motegilab/ai-orchestrator-loop-claude）に改善が加わったとき、自分のPJにも取り込む方法です。

💡
GitHub の「Use this template」で作ったリポジトリはフォークではないため、自動で更新は届きません。手動で取り込む必要があります。
エクスプローラーのアドレスバーからコピー & ペーストできます。入力するとコマンドが自動更新されます。

① upstream を一度だけ登録する

プロジェクトフォルダで一度だけ実行します（2回目以降は不要）。

git remote add upstream https://github.com/motegilab/ai-orchestrator-loop-claude.git
git remote -v   # 確認

② 取り込み方を選ぶ

パターンこんな時に使うリスク
A. 特定ファイルだけ取得 hooks や scripts の一部だけ更新したい 低（他は変わらない）
B. まるごとマージ 大きな改善を全部取り込みたい 中（コンフリクトが出る場合あり）

パターン A — 特定ファイルだけ取得（おすすめ）

よく更新される hooks・scripts・マニュアルだけを選んで取り込みます。SSOT.md や milestones.json は上書きされません。

git fetch upstream

# ── Hooks（全部まとめて。新Hook追加にも自動対応）
git checkout upstream/main -- .claude/hooks/

# ── スクリプト（バグ修正・機能追加）
git checkout upstream/main -- tools/scripts/loop_run.py
git checkout upstream/main -- tools/scripts/ssot_check.py

# ── マニュアル（手順・トラブル対処の最新版）
git checkout upstream/main -- docs/setup-guide.html
git checkout upstream/main -- docs/onboarding-prompt.md
git checkout upstream/main -- docs/ssot-setup-prompt.md

# ── Skills（新Skillが追加されていた場合）
git checkout upstream/main -- .claude/skills/

# 取り込んだ内容を確認してコミット
git diff --stat
git commit -m "chore: sync hooks, scripts, docs from upstream template"

パターン B — まるごとマージ

テンプレートの全変更をまとめて取り込みます。自分のPJに変更がある場合はコンフリクトが発生することがあります。

⚠️
merge の前に必ず未コミットの変更をコミットしてください。
未コミットのファイルがあると error: Your local changes would be overwritten by merge が出て止まります。
# ① 未コミットの変更を先にコミット（重要）
git add -A
git commit -m "chore: commit local state before upstream merge"

# ② upstream を取得してマージ
git fetch upstream
git merge upstream/main --allow-unrelated-histories

# ③ マージ後に残ったファイルをコミット
git add -A
git commit -m "chore: upstream template をマージ、オーケストレーター層を追加"

# ④ push
git push

⚠️ merge でロックエラーが出たとき

以下のどちらかのエラーが出た場合は、git のロックファイルが残っているのが原因です。

fatal: update_ref failed for ref 'ORIG_HEAD': cannot lock ref 'ORIG_HEAD':
Unable to create '.../.git/ORIG_HEAD.lock': File exists.

または

error: Unable to create '.../.git/packed-refs.lock': File exists.

対応するロックファイルを削除してから merge を再実行してください。

# 該当するファイルを削除（どちらか or 両方）
Remove-Item ".git\ORIG_HEAD.lock"   -Force -ErrorAction SilentlyContinue
Remove-Item ".git\packed-refs.lock" -Force -ErrorAction SilentlyContinue

# merge を再実行
git merge upstream/main --allow-unrelated-histories
💡
VSCode や別のターミナルが同じフォルダで git 操作した後に残るファイルです。害はなく削除して問題ありません。警告が出ても最後に [main xxxxxxx] と表示されれば成功です。

コンフリクトが出たときの対処

マージ後にコンフリクトが出た場合、以下のファイルは自分側を優先して解決してください。

ファイル理由
SSOT.mdPJ固有の設計を含むため、自分の内容を残す
CLAUDE.mdPJ固有の指示を含むため、自分の内容を残す
tasks/milestones.json進行中のタスクが入っているため、自分の内容を残す
policy/ssot_integrity.jsonSSOT ハッシュが記録されているため、自分の内容を残す
# コンフリクトを「自分側で解決」するコマンド例
git checkout --ours SSOT.md
git checkout --ours CLAUDE.md
git checkout --ours tasks/milestones.json
git checkout --ours policy/ssot_integrity.json
git add SSOT.md CLAUDE.md tasks/milestones.json policy/ssot_integrity.json
git commit -m "chore: merge upstream, keep project-specific files"
